## Summary

Kotlin/JVM emits `.class` files containing JVM bytecode—stack-based instructions executed by the JVM.

## Mental model

Bytecode is the portable machine language of the JVM. Kotlin features like inline functions, coroutines, and lambdas have defined lowering strategies into bytecode patterns.

## Example

```bash
kotlinc Hello.kt -include-runtime -d hello.jar
javap -c HelloKt
```

## Common mistakes

- Equating Kotlin source structure 1:1 with generated classes
- Surprising yourself with synthetic accessors for internal visibility
- Debugging without inspecting generated bytecode for interop issues
